#  4. Conditional Statements

##  If Statement

Executes code when condition is true:

```javascript
if (condition) {
  // code
}
```

##  Nested If

An `if` inside another `if`

##  Else Statement

Executes when condition is false:

```javascript
if (condition) {
  // true block
} else {
  // false block
}
```

##  Switch Case

Used for multiple conditions:

```javascript
switch (x) {
  case 1:
    break;
}
```

Synchronous program:
    -> code executes in a sequential order (line by line), next line waits for the current line to finish execution before running
    -> disadvantage is if one block/line takes more time to finish then all the after blocks/lines will wait so its not efficient

To solve this we have 3 ways:
    1-> callbacks: its a function that runs after something finishes, callbacks can be any type of functions(generic function,arrow function, etc)
    this way will be hectic to debug or to understand if we have multiple callbacks and this is called callback hell
    
    => setTimeout(callback, time) -> will execute the callback function once after a specific delay

    => setInterval(callback, time) -> will execute the callback function repeatedly every specific interval until it stopped

    => clearInterval(setInterval) -> stops a setInterval from running, we should store the interval in a variable first to clear it
    
    2-> Promise: an object in js which introduced to solve callback hell and it represents a value that will be available in future
        there are 3 states in Promise
            Pending -> initial state
            Resolve -> operation successfull then promise holds result value
            Reject -> operation failure then promise holds error message

//async function
- A function declared with `async` that always returns a Promise. 
- Inside an `async` function, you can use `await`. The function pauses at `await` but doesn't block the whole program.

//await
- An operator you can only use inside `async` functions. 
- Pauses the `async` function until a Promise settles, then returns the result. 
- Makes async code look sequential. If the Promise rejects, `await` throws an error you can catch with `try/catch`.

//OOPS 
    -> object oriented programing which makes code to be organised and more readable

Four principles  of OOPS:
    -> Encapsulation
        => securing the internal structure wihtout giving direct access it gives controlled access by using access modifiers
    -> Inheritence
        => we can create classes which can extends from another class using extends keyword
        => so child class can inherite properties/methods from parent class
    -> Polymorphism
        => one thing having many forms, like one method from one class can be having different behaviours for each class
    -> Abstraction
        => hiding the complex internal implementation and showing what is neccesary






Generally the objects would get thier own copies of functions. but this leads to more memory usage as same function copy is created several times whenever the object is created. and this lead to ineffitient usage of memory.
So to use the memory effitiently we use something called as "prototype".

Prototype:
    -> it is used to generate a single copy of a function and all the objects will use the same copy.
    -> this makes the memory usage effitient.

callback function
    -> a function which has been called by a function
higher order function
    -> a function which calls a callback function

stack : 
    ->it stores data in last in first out order (LIFO)
    -> premitive types will be stored in stack and it fetches fastly 
    -> primitive copy

heap: 
    -> it stores objects like funtions, arrays and in random order and its fetches slowly compared to stack
    -> refrence copy
